Why Power Pages? The Strategic Choice

Platform Decision Rationale

  • Anonymous Access Critical: Citizens won't create accounts to report potholes - removing friction increases adoption by 10x
  • Cost Predictability: Flat-rate licensing vs per-transaction costs that could spiral with 300k residents
  • Existing Microsoft Investment: Leverages current M365/Azure spend rather than new supplier contracts
  • Low-Code Maintainability: Visual designers mean council staff can make changes without supplier dependency

Integration Risk Mitigation

High-Risk: Dynamics 365 CRM

Risk: Data conflicts in bi-directional sync

Mitigation:

  • Implement "last write wins" with audit trail
  • API throttling (6,000 requests per 5 minutes)
  • Queue changes in Dataverse for batch retry

High-Risk: Confirm Asset Management

Risk: Legacy SOAP, 30-second timeouts

Mitigation:

  • Build abstraction layer with transformation maps
  • Implement circuit breaker pattern
  • Email fallback with CSV attachment

Medium-Risk: SAP Finance

Risk: File format changes, SFTP issues

Mitigation:

  • Schema validation on every transfer
  • File versioning and 30-day retention
  • Alert on failed transfers

What Requires Developer Support

⚠️ Pro-Code Requirements (Specialist Support Needed)

  • Custom Connectors: Integration with systems lacking standard connectors
  • Azure Functions: Complex data transformations, external API calls
